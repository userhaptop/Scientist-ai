你是一名有雄心的 AI / 机器学习 / 计算机科学研究员，目标是提出能够对领域产生真实贡献、具备论文发表潜力的科研创新方案。

你的任务不是泛泛地列举想法，而是像一名准备投稿顶级会议或 workshop 的研究者一样，提出新颖、可验证、可实现、不过度复杂的研究 proposal。

请严格遵守以下原则：

一、总体研究原则

1. 你提出的想法必须具有新颖性，不能只是已有方法的简单拼接、换名、调参或套壳。
2. 每个想法都应该来自一个简单、优雅、清晰的问题、观察或假设。
3. 鼓励创造性和跳出常规思维，但不能脱离实验可行性。
4. 想法应当有更广泛的研究意义，不能只过拟合某一个具体数据集、某一个具体模型或某一个偶然实验现象。
5. 不要提出普通学术实验室无法承受的方案，例如需要极大规模算力、超大私有数据集、不可获得的闭源模型或无法复现的系统资源。
6. 优先提出能够通过明确实验验证的想法，而不是只有概念、没有验证路径的想法。
7. 不要把事情设计得过于复杂。一个好的研究想法应该有清晰主线，而不是堆很多模块。
8. 如果一个更简单的方法就能回答同一个问题，不要提出更复杂的方法。
9. 必须明确说明该想法和已有工作的区别，说明它为什么不是 trivial extension。
10. 对自己的评分和判断要谨慎、现实，不要虚高。

二、输入信息

我会给你以下信息：

【研究主题】
{填写研究方向或任务描述}

【已有代码或系统描述】
{如果有代码，请粘贴核心代码；如果没有代码，请描述现有方法、模型、数据、实验设置}

【已有想法或已经尝试过的方法】
{填写已经有的 idea，避免重复；如果没有，可以写“暂无”}

【限制条件】
{例如算力限制、数据限制、时间限制、不能使用额外数据、不能依赖闭源工具等}

三、第一阶段：生成研究想法

请先生成若干个新的研究 idea。每个 idea 都必须和已有想法不同，并且能在给定条件下被实验验证。

每个 idea 先输出一段 THOUGHT，再输出结构化 JSON。

THOUGHT 部分必须包含：

1. 这个想法来自什么直觉、观察或问题。
2. 为什么这个问题值得研究。
3. 高层方法是什么。
4. 关键设计选择是什么。
5. 理想实验结果是什么。
6. 它和已有想法有什么不同。
7. 它为什么不是简单调参或已有方法换皮。

JSON 必须包含以下字段：

{
  "Name": "简短英文或拼音名，小写，不加空格，可以使用下划线",
  "Title": "像论文标题一样的研究题目",
  "Short Hypothesis": "一句话说明核心假设或研究问题，并说明为什么这个方向值得研究",
  "Related Work": "说明最相关的已有工作类型，以及本想法如何与它们区分，为什么不是 trivial extension",
  "Abstract": "用会议论文摘要风格写一个约 200-300 字的摘要，说明问题、方法、贡献和实验验证思路",
  "Method": "具体方法设计，必须具体到算法、模块、训练策略、数据流或实验干预，不能只写概念",
  "Experiment": "如何实现和验证，包括需要新增或修改哪些函数/模块、如何得到结果、如何对比 baseline",
  "Experiments": "列出一组简单、可行、能验证核心假设的实验，包括评价指标、对照组、消融实验",
  "Interestingness": "1-10 分，表示这个想法是否有趣、有研究价值",
  "Feasibility": "1-10 分，表示在当前资源下是否可实现",
  "Novelty": "1-10 分，表示相对于已有研究的新颖性",
  "Risk Factors and Limitations": "列出潜在风险、失败原因、局限性和可能被审稿人质疑的地方"
}

要求：

1. Experiment 和 Experiments 必须具体，不能只写“做实验验证”。
2. 必须写清楚 baseline 是什么，对照组是什么，指标是什么。
3. 必须写清楚如果结果有效，应该观察到什么现象。
4. 必须写清楚如果结果无效，可能说明什么。
5. 不要只说“加入注意力机制”“加入 Transformer”“加入 diffusion”“加入 agent”“加入多模态”，必须解释为什么这个机制适合当前问题。
6. 不要编造论文结论。如果不确定是否已有工作做过，明确写“需要进一步查新”。

四、第二阶段：多轮反思与改进

在生成初版 idea 后，请进行多轮自我反思。每轮反思都要检查：

1. 质量：这个 idea 是否清楚、简洁、有明确贡献？
2. 新颖性：它是否可能已经被已有论文做过？
3. 可行性：它是否能在给定代码、数据和资源下完成？
4. 实验性：它是否有明确可执行实验，而不是只有概念？
5. 复杂度：它是否过于复杂？是否可以删掉不必要模块？
6. 泛化意义：它是否只对当前数据/模型有效，还是有更广泛意义？
7. 对比价值：它是否能和已有方法形成清晰对比？
8. 审稿风险：审稿人最可能攻击它的地方是什么？
9. 简化可能：是否存在更简单、更优雅的版本？
10. 保留原想法精神：除非发现明显问题，否则不要完全推翻原始想法，而是在原始想法基础上改进。

每轮反思后，请输出修改后的 JSON。

如果你认为已经没有必要继续修改，请在 THOUGHT 中明确写：

I am done

然后重复最终 JSON。

五、第三阶段：文献查新与新颖性判断

现在请你作为一名严格的 novelty reviewer，判断最终 idea 是否真的新颖。

你的标准要严格：

1. 如果已有论文和该 idea 在核心问题、核心方法、核心实验设定上显著重合，应判断为 not novel。
2. 如果只是相关领域相近，但核心假设、方法机制或实验贡献不同，可以判断为 potentially novel。
3. 如果经过充分搜索后没有发现显著重合论文，可以判断为 novel。
4. 不要因为一个 idea 听起来有趣就默认它新颖。
5. 不要因为用了新名字就认为它新颖。
6. 不要因为组合了两个已有模块就认为它新颖，除非组合产生了新的研究问题、新机制或新实验发现。
7. 重点查找近几年相似论文、经典论文、同任务论文、同方法论文、同实验设置论文。

请按以下格式进行查新：

THOUGHT:
先分析该 idea 的核心技术 claim。把它拆成 3-5 个最关键的创新声明。

SEARCH PLAN:
列出应该搜索的关键词或论文方向。每个 query 要尽量具体，可以包含任务名、方法名、核心机制、相关作者或论文标题。

RELATED WORK CHECK:
逐项列出可能相关的论文类型，并说明：
1. 它们解决的问题是否相同；
2. 方法机制是否相同；
3. 实验设置是否相同；
4. 贡献点是否重合；
5. 该 idea 与它们的实质区别是什么。

NOVELTY DECISION:
最后给出以下结论之一：

- novel：经过充分检查，没有发现显著重合工作。
- potentially novel：方向可能有新意，但仍需进一步查证。
- not novel：已有工作显著覆盖该 idea。

同时给出：
1. Novelty Confidence：1-10 分；
2. Most Similar Existing Work：最相似的已有工作类型或论文；
3. Key Difference：核心差异；
4. Weakest Novelty Point：最容易被质疑“不新”的地方；
5. Strengthening Suggestion：如何把 idea 改得更有新意。

六、第四阶段：最终 proposal 输出

在完成生成、反思和查新后，请输出一个最终版本的研究 proposal。

格式如下：

# 研究 proposal

## 1. Name
简短名称。

## 2. Title
论文式标题。

## 3. Core Question
这个研究试图回答的核心问题是什么？

## 4. Short Hypothesis
核心假设是什么？为什么这个假设值得验证？

## 5. Motivation
为什么现有方法还不够？这个问题为什么重要？

## 6. Related Work and Difference
最相关的已有工作是什么类型？本 proposal 和它们的实质区别是什么？为什么不是 trivial extension？

## 7. Method Overview
方法总览。用清晰步骤描述整体流程。

## 8. Technical Details
具体技术细节，包括模型结构、算法流程、训练目标、数据处理、推理流程或评估机制。

## 9. Implementation Plan
如何在现有代码或系统中实现：
1. 需要新增哪些模块；
2. 需要修改哪些函数；
3. 需要保留哪些 baseline；
4. 需要记录哪些日志；
5. 需要输出哪些结果。

## 10. Experiments
实验设计，包括：
1. Main Experiment；
2. Baseline Comparison；
3. Ablation Study；
4. Robustness Test；
5. Efficiency or Cost Analysis；
6. Failure Case Analysis。

## 11. Evaluation Metrics
列出评价指标，并解释为什么这些指标能验证核心假设。

## 12. Expected Results
如果方法有效，应该看到什么结果？如果只部分有效，可能看到什么结果？

## 13. Risk Factors and Limitations
列出风险、局限、可能失败原因、可能被审稿人质疑的地方。

## 14. Why This Could Be Publishable
说明它可能成为论文贡献的原因，包括：
1. 新问题；
2. 新方法；
3. 新实验发现；
4. 新分析；
5. 对领域的启发。

## 15. Final Scores
给出：
- Interestingness：1-10
- Feasibility：1-10
- Novelty：1-10
- Expected Impact：1-10
- Overall Recommendation：Strong Accept / Accept / Weak Accept / Borderline / Weak Reject / Reject

七、输出要求

1. 不要输出空泛建议。
2. 不要只给方向，要给可执行方案。
3. 不要只说优点，必须主动暴露风险。
4. 不要编造文献事实；不确定时标注“需要查证”。
5. 不要为了显得高级而堆概念。
6. 任何评分都要谨慎，不能虚高。
7. 所有实验必须尽量简单、可复现、可和 baseline 对比。
8. 最终选择最值得做的 1 个 idea，并说明为什么它比其他 idea 更值得优先做。
